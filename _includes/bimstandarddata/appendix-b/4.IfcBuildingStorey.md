# IfcBuildingStorey

The building storey has an elevation and typically represents a (nearly) horizontal aggregation of spaces that are vertically bound.
Note: Occupancy Number should describe total calculated number of occupants in the building

| Name | PropertySet | Description | User Input (YES/NO) | DataType | DataSample | IsRequired? |
| --- | --- | --- | --- | --- | --- | --- |
| GrossHeight | Qto_BuildingStoreyBaseQuantities | IFC Standard Property. | No | double | 1 | Yes |
| NetHeigtht | Qto_BuildingStoreyBaseQuantities | IFC Standard Property. | No | double | 1 | Yes |
| TotalBuildupArea | Building Permit | Total buildup area of the building storey as defined by the building code. Only required in the AR models. | Yes | double | 1 | Yes |
| TotalGrossArea | Building Permit | Total gross area of the building storey as defined by the building code. Only required in the AR models. | Yes | double | 1 | Yes |
| TotalFloorGrossArea | Building Permit | Total gross floor area of the building storey as defined by the building code. Only required in the AR models. | Yes | double | 1 | Yes |
| TotalNetArea | Building Permit | Total net area of the building storey as defined by the building code. Only required in the AR models. | Yes | double | 1 | Yes |
| BuildingPartNum | Building Permit | A number to assign a storey to a building part. Only required when the building is complex and has multiple parts. E.g. podium and two tower.

When the BIM model contains building parts (e.g. two towers) that share levels, each building part should be exported in a separate IFC file so different building part numbers can be assigned to same levels. | Yes | double | 1 | Conditional |
